Implementation: Outlier Detection¶

Detecting outliers in the data is extremely important in the data preprocessing step of any analysis. The presence of outliers can often skew results which take into consideration these data points. There are many "rules of thumb" for what constitutes an outlier in a dataset. Here, we will use Tukey's Method for identfying outliers: An outlier step is calculated as 1.5 times the interquartile range (IQR). A data point with a feature that is beyond an outlier step outside of the IQR for that feature is considered abnormal.

In the code block below, you will need to implement the following:

  • Assign the value of the 25th percentile for the given feature to Q1. Use np.percentile for this.
  • Assign the value of the 75th percentile for the given feature to Q3. Again, use np.percentile.
  • Assign the calculation of an outlier step for the given feature to step.
  • Optionally remove data points from the dataset by adding indices to the outliers list.

NOTE: If you choose to remove any outliers, ensure that the sample data does not contain any of these points!
Once you have performed this implementation, the dataset will be stored in the variable good_data.

Answer: The wholesale distributor can make use of the customer segments discovered by clustering to uncover the types of customers present and their underlying purchasing behaviour. By looking at such underlying purchasing behaviour, the wholesaler can make an informed decision on whether changing the frequency of their delivery would have an impact on their customers.

For example, customers in segment 0 are retailers that tend to move products quickly. If you decrease your delivery but don't increase the volume of each delivery, then the retailers won't be happy if they run out of stock in between the deliveries.

This decision also depends on the product categories that the customers buy. Some products such as fresh and milk will have a shorter shelf life, so if the wholesaler makes less frequent deliveries, that could lead to their products not being up to standards. Conversely, if the products are detergents_paper, making less frequent deliveries probably would not matter at all.

Overall, knowing the customer segments and their buying behaviour the wholesaler can be much better informed when making these decisions. Based on the customer segments present, I believe segment 1 will react more positively and both segments 0 and 2 will react negatively. This is because for segment 1, they would not mind having less frequent deliveries as the speed on which they turnover their products would be slower that segments 0 and 2. Obviously there may be other factors at play but having these segments allows us to see these factors and test them accordingly.

How can A/B test be done in this case?

For A/B tests to be effective, the experiment group has to be highly similar to the control group. This is where the segmentation work that we did becomes useful. For each segment, we can run an A/B test where you select 10-15 customers as a representative sample population, you split them into a control group and experiemnt group, roughly 50% in each group, then run the test for the experiemnent group. You then have an objective measure of whether the experiment group reacted more positively or negtively to the control group. We must also be sure that for the experiment group we only change one variable, in our case it is the frequency of weekly delivery.

I believe running the A/B test once for each segment is not sufficient. I think 2-3 samples from each segment selected to run the tests and we will get reliable results for that segment. So in total I think 9-10 A/B tests would be obtained in order to accurately gauge the effectiveness of the experiment.
